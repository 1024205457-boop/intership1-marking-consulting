# AI 辅助市场研究工作流 — 方法论

## 1. 问题定义

传统市场咨询项目中，以下环节最耗时：

| 环节 | 典型耗时 | 痛点 |
|------|----------|------|
| Brief 澄清 | 1–2 天 | 需求模糊、研究问题不可验证 |
| 数据采集与清洗 | 3–5 天 | 人工爬取/整理，重复劳动 |
| 定性归纳 | 2–3 天 | 分析师主观性强，难以结构化 |
| 可信度验证 | 1–2 天 | 洞察缺乏原声支撑，客户质疑 |
| 报告撰写 | 2–3 天 | PPT 制作周期长，格式不统一 |

**目标**：将「Brief → 洞察报告」压缩 50%+，同时保证洞察可追溯、可验证。

## 2. 研究场景

**品类**：IMF 婴幼儿配方奶粉

**客户需求**：
- 洞察电商消费者购物旅程与核心触点
- 理解不同决策模式人群（深度学习型/适度研究型/快速决策型）的行为差异
- 识别品牌在旅程各环节的转化机会

**数据来源**：
- 社媒舆情（小红书/抖音/知乎/微博）
- 电商评论（天猫/京东）
- 社群口碑（妈妈群）

## 3. Pipeline 架构

```
Brief 拆解 → 数据采集 → 框架分析 → 三元校验 → 报告交付
```

5 个 stage 均为纯函数 + 标准 IO，可独立测试和替换。

## 4. 框架化分析

对齐消费者旅程框架（Aware → Light I → Deep I → Purchase），输出四模块结构化洞察：

1. **人群分层**：基于决策行为识别深度学习型(22%)/适度研究型(41%)/快速决策型(37%)
2. **触点效率**：各平台在不同旅程阶段的角色（抖音破圈/小红书深度种草/电商转化）
3. **决策路径**：从品牌认知到首购/转牌/复购的关键节点与转化率
4. **转化建议**：基于触点效率和路径瓶颈输出可行动的媒介策略建议

## 5. 三元校验

每条洞察必须通过三重检验：

### 源校验（Source Check）

- 洞察是否绑定了有效的 `source_id`
- 来源记录是否存在于原始数据集
- 主题关键词是否与原声有语义关联

### 交叉校验（Cross Check）

- 同一主题的多条原声情感方向是否一致
- 避免基于单条极端样本得出泛化结论

### 逻辑校验（Logic Check）

- 检测「100%」「所有用户」「必然」等过度推断
- 可选 LLM 辅助判断推理链是否跳跃

### 置信度分级

| 等级 | 分数 | 处理方式 |
|------|------|----------|
| high | ≥ 0.75 | 直接进入报告 |
| medium | 0.5–0.75 | 进入报告，标注置信度 |
| low | < 0.5 | 进入人工复核队列 |

## 6. 技术选型

| 层 | 选择 | 理由 |
|----|------|------|
| 数据接入 | Adapter 接口 | 可扩展爬虫/API/行为监测数据 |
| 分析引擎 | 规则 + 关键词 | 确定性高，可解释 |
| LLM | 可选增强 | 逻辑校验辅助，非必须 |
| 报告 | Jinja2 HTML | 轻量、可定制、不依赖 Office |

## 7. 局限性

- 当前 Demo 使用 65 条模拟舆情，非真实行为监测数据
- 分析以规则引擎为主，LLM 为可选增强
- 人群分层基于关键词匹配，真实项目需结合行为监测数据
